4. ТРИ РЕЖИМА РАБОТЫ:

   - Подтверждение успешного бронирования

   - Управление специалистами (CRUD операции)

   - Загрузка бронирований через GET /api/specialists/:id/bookings

   - Детали брони с учётом часовых поясов
   - Счётчик новых броней (WHERE created_at > last_login)

8. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ BACKEND:
   - Node.js + Express сервер
   - Подключение к PostgreSQL через pg или pg-promise
   - Валидация данных на сервере
   - Обработка ошибок и возврат понятных HTTP статусов

   - Транзакции для критических операций
